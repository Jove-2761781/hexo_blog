# Using dbms_file_transfer(DFT)

All steps in this procedure are run as the oracle user that is a member of the OSDBA group. OS authentication is used to connect to both the source and destination databases.

  1. Initial Setup

    1.1 创建 xtts 目录，解压 rman_xttconvert_v3.zip 软件包

    1.2 配置 xtt.properties 参数

    1.3 （可选）目标端配置临时实例用于增量备份恢复，目标端 11.2.0.4 及以上跳过此步骤

    1.4 目标端创建连接源端的 dblink

    1.5 创建数据库目录

    1.6 源端和目标端设置临时目录

  2. Prepare Phase for dbms_file_transfer Method

    2.1 源端执行以下命令，创建 getfile.sql 和 xttnewdatafiles.txt 文件

    2.2 拷贝 getfile.sql 和 xttnewdatafiles.txt 文件文件至目标端

    2.3 目标端进行数据文件拷贝

  3. Roll Forward Phase

    3.1 源端生成增量备份文件至 backupformat

    3.2 拷贝增量备份文件、xttplan、tsbkupmap 至目标端 stageondest

    3.3 目标端做增量前滚

    3.4 源端确定下一个增量备份的 FROM_SCN

    3.5 Repeat the Roll Forward Phase (Phase 3)

  4. Transport Phase

    4.1 源端将表空间置为只读模式

    4.2 重复步骤 3.1 - 3.3，做最后一次增量前滚

  5. Import Object Metadata into Destination Database

    5.1 目标端创建业务用户

    5.2 目标端生成 Data Pump TTS 命令

    5.3 迁移表空间元数据

    5.4 迁移存储过程、触发器、函数、包、视图、序列

    5.5 导入同义词、dblink 和其他对象

  6. Post-migration

    6.1 检查数据库一致性

    6.2 业务用户修改权限和默认表空间

    6.3 表空间置为读写模式

    6.4 校验数据文件完整性

    6.5 收集统计信息
