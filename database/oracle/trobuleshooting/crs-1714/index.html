# CRS-1714

数据库：Oracle RAC 11.2.0.4

操作系统：OracleLinux 7u9

磁盘采用的是 iSCSI + ASMLib

# Case

重启服务器的时候出现一个奇怪的现象，crs 无法正常启动

查看 /u01/app/11.2.0/grid/log/rac1/alertrac1.log 日志，发现无法识别到 votedisk

检查 oracleasm configure，已经设置开机启动扫描磁盘

而且手动扫描磁盘是可以扫到磁盘的，说明配置是没啥问题的

那么问题出在哪呢？

# Solution

经过排查可能是 oracleasm.service 的服务比 iscsid.service 的服务更早启动，导致没有扫描到 iscsi 磁盘，所以修改一下服务让 oracleasm 在 iscsid 后面启动

重启后，问题解决。
