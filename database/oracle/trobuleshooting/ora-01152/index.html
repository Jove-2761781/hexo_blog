# ORA-01152

<font color=error>ORA-01152: file 1 was not restored from a sufficiently old backup</font>

在用 rman 恢复数据库时当执行到将数据库 open 阶段遇到 ORA-01152 错误

# Cause

由于 controlfile 里记录的 SCN 与 datafile 里的不一致导致。比如通过 restore 出来的 controlfile 的 SCN 比 datafile 记录的要小。解决方法有两条：

  1. 以 datafile 的 SCN 为准，将 controlfile 恢复到与其一致，此时就会报 ORA-01152 的错误，然后我们通过 recover database，进而利用归档日志来完成 controlfile 和 datafile 的 SCN 一致，归档找不到再去利用 redolog 来恢复。

  2. 通过更改内置参数跳过一致性的检查启动数据库，不过可能发生 ORA-00600 [2662] 错误。
